# SupportNaija

Transparent support for Nigerians. Donate from ₦500. Beneficiaries are BVN-verified
(with optional NIN), pool balances are public, and disbursements are tracked.

Built with Next.js 16 (App Router), React 19, Tailwind v4, Convex, and Clerk.

## Tech stack

- **Next.js 16** — App Router, server components, file-based routing.
- **React 19** — concurrent features, async server components.
- **Tailwind v4** — design tokens via CSS variables in `src/app/globals.css`.
- **Convex** — typed backend (queries, mutations, scheduled functions) under
  `convex/`. Generated types live in `convex/_generated/`.
- **Clerk** — auth provider wrapping the app via `<ClerkProvider>` in
  `src/app/layout.tsx`.
- **Paystack** *(planned)* — donation checkout + webhook → flips a pending
  donation to `succeeded`.
- **Termii** *(planned)* — airtime disbursements for verified beneficiaries.

## Running locally

1. **Install deps**

   ```bash
   npm install
   ```

2. **Configure env**

   ```bash
   cp .env.example .env.local
   ```

   Fill in at minimum:
   - `NEXT_PUBLIC_CONVEX_URL` (set automatically by `npx convex dev`)
   - `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` and `CLERK_SECRET_KEY`

3. **Start Convex**

   In one terminal:

   ```bash
   npx convex dev
   ```

   This pushes the functions in `convex/` to your Convex deployment and writes
   `NEXT_PUBLIC_CONVEX_URL` into `.env.local` for you.

4. **Start Next.js**

   In another terminal:

   ```bash
   npm run dev
   ```

   Open <http://localhost:3000>.

## Deploying

### Convex

Push your prod deployment:

```bash
npx convex deploy
```

Set server-side secrets on the Convex deployment (these are NOT exposed to the
browser):

```bash
npx convex env set PAYSTACK_SECRET_KEY sk_live_xxx
npx convex env set PAYSTACK_WEBHOOK_SECRET whsec_xxx
npx convex env set TERMII_API_KEY tl_xxx
npx convex env set VERIFICATION_API_KEY xxx
npx convex env set VERIFICATION_HASH_SALT "$(openssl rand -hex 32)"
```

### Vercel (or similar)

Set the public + server vars from `.env.example` in your hosting provider's env
config. The browser only sees variables prefixed with `NEXT_PUBLIC_`.

```
NEXT_PUBLIC_CONVEX_URL=https://your-prod.convex.cloud
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_live_xxx
CLERK_SECRET_KEY=sk_live_xxx
NEXT_PUBLIC_PAYSTACK_PUBLIC_KEY=pk_live_xxx
NEXT_PUBLIC_SITE_URL=https://supportnaija.com
```

Then:

```bash
npm run build
```

## What's stubbed for the MVP

- **Paystack handover.** `convex/donations.ts` creates a `pending` donation with
  a generated reference. Wire up the Paystack inline widget on the receipt
  screen and a Convex HTTP route to verify webhooks (`x-paystack-signature`)
  and flip status to `succeeded`.
- **BVN / NIN verification.** The apply form collects + validates input shape.
  Move the network call into a Convex action keyed by `VERIFICATION_PROVIDER`,
  hash with `VERIFICATION_HASH_SALT`, and write to `recipients`.
- **Proof file uploads.** Stored in component state for now; add a Convex
  storage upload (`ctx.storage.generateUploadUrl()`) and store URLs on the
  application doc.
